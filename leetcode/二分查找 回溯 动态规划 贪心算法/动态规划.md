# **01. 编辑距离问题**

> **1.1 编辑距离理解**
>
> > [!important] **[Hello 算法 - 编辑距离问题](https://www.hello-algo.com/chapter_dynamic_programming/edit_distance_problem/)**
>
> - 编辑距离，也称 Levenshtein 距离，指两个字符串之间互相转换的最少修改次数。通常用于在信息检索和自然语言处理中度量两个序列的相似度

> _**问题描述**_
>
> - 输入两个字符串 s 和 t，返回将 s 转换为 t 所需的最少编辑步数。你可以在一个字符串中进行三种编辑操作：插入一个字符、删除一个字符、将字符替换为任意一个字符。
> 
> - _**示例 ⇒ 1.**_ 将 **kitten** 转换为 **sitting** 需要编辑 **3** 步，包括 **2** 次替换操作与 **1** 次添加操作。_**2.**_ 将 **hello** 转换为 **algo** 需要 **3** 步，包括 **2** 次替换操作和 **1** 次删除操作。
> 
> ![[动态规划 - 1.png|Untitled 87.png]]

> _**编辑距离问题可以很自然地用 决策树模型 来解释**_
>
> - 字符串对应树节点，一轮决策（一次编辑操作）对应树的一条边
> 
> - 在不限制操作的情况下，每个节点都可以派生出许多条边，每条边对应一种操作，这意味着从 `hello` 转换到 `algo` 有许多种可能的路径。
> 
> - 从决策树的角度看，本题的目标是求解节点 `hello` 和节点 `algo` 之间的最短路径
> 
> ![[动态规划 - 2.png|Untitled 1 58.png]]

> _**第一步：思考每轮的决策，定义状态，从而得到 dp 表**_
>
> - 每一轮的决策是对字符串 _**s**_ 进行一次 编辑操作
> 
> - 设字符串 _**s**_ 和 _**t**_ 的长度分别为 _**n**_ 和 _**m**_，我们先考虑两字符串尾部的字符`**s[n-1]**`和`**t[m-1]**`
> 
> - 若`**s[n-1]**`和`**t[m-1]**`相同，我们可以跳过它们，直接考虑`**s[n-2]**`和`**t[m-2]**`
> 
> - 若`**s[n-1]**`和`**t[m-1]**`不同，我们需要对 _**s**_ 进行一次编辑（插入、删除、替换），使得两字符串尾部的字符相同，从而可以跳过它们，考虑规模更小的问题
> 
> - 也就是说，在字符串 _**s**_ 中进行的每一轮决策，都会使得 _**s**_ 和 _**t**_ 中剩余的待匹配字符发生变化
> 
> - 因此，状态为当前在 _**s**_ 和 _**t**_ 中考虑的第 _**i**_ 和第 _**j**_ 个字符，记为==`**[i,j]**`==
> 
> - 状态`**[i,j]**`对应的子问题 ⇒ 将 _**s**_ 的前 _**i**_ 个字符更改为 _**t**_ 的前 _**j**_ 个字符所需的最少编辑步数
>     
>     - 得到一个尺寸为==`**(i+1) × (j+1)**`==的二维 _**dp**_
>     

> _**第二步：找出最优子结构，进而推导出状态转移方程**_
>
> - 考虑子问题 `dp[i,j]`，其对应的两个字符串的尾部字符为 `s[i-1]` 和 ` t[j-1]`
> 
> - 情况 1 ⇒ 在 `s[i-1]` 之后添加 `t[j-1]`，则剩余子问题 `dp[i,j-1]`
> 
> - 情况 2 ⇒ 删除 `s[i-1]`，则剩余子问题 `dp[i-1,j]`
> 
> - 情况 3 ⇒ 将 `s[i-1]` 替换为 `t[j-1]`，则剩余子问题 `**dp[i-1,j-1]**`
> 
> - _**最优子解构 ⇒**_ `dp[i,j]` 的最少编辑步数等于 `dp[i,j-1]、dp[i-1,j]、dp[i-1,j-1]` 三者中的最少编辑步数，再加上本次的编辑步数 _**1**_。即 `dp[i,j] = min(dp[i,j-1], dp[i-1,j], dp[i-1,j-1])+1`
> 
> - 特殊情况 ⇒ 当 `s[i-1]` 和 `t[j-1]` 相同时，无须编辑当前字符，其状态转移方程为 `dp[i,j] = dp[i-1,j-1]`
> 
> ![[动态规划 - 21.png|Untitled 2 54.png]]

> _**第三步：确定边界条件和状态转移顺序**_
>
> - 当两字符串都为空时，编辑步数为 _**0**_，即`**dp[0,0]=0**`
> 
> - 当 _**s**_ 为空但 _**t**_ 不为空时，最少编辑步数等于 _**t**_ 的长度，即首行`**dp[0,j]=j**`
> 
> - 当 _**s**_ 不为空但 _**t**_ 为空时，最少编辑步数等于 _**s**_ 的长度，即首列`**dp[i,0]=i**`
> 
> - `**dp[i,j]**`依赖 左方、上方、左上方的解，因此通过两层循环正序遍历整个 _**dp**_ 表即可

> _**代码详解**_
>
> ```JavaScript
> var minDistance = function(word1, word2) {
>   // dp[i][j]表示word1的前i个字符变成word2的前j个字符的最少操作数
>   const dp = Array.from({ length: word1.length + 1 }, () => Array(word2.length + 1).fill(Infinity))
>   dp[0][0] = 0
>   // 初始化
>   for (let col = 1; col < dp[0].length; col++) dp[0][col] = col
>   for (let row = 1; row < dp.length; row++) dp[row][0] = row
> 
>   for (let i = 1; i < dp.length; i++) {
>     for (let j = 1; j < dp[0].length; j++) {
> 	    // 如果word1的第i个字符和word2的第j个字符相等，则不用添加操作步数
>       if (word1[i - 1] === word2[j - 1]) dp[i][j] = dp[i - 1][j - 1]
>       else dp[i][j] = Math.min(dp[i - 1][j - 1], dp[i - 1][j], dp[i][j - 1]) + 1
>     }
>   }
>   return dp[word1.length][word2.length]
> };
> ```

# **02. 单词拆分**

> _**问题描述 & 代码详解**_
>
> > [!important] **[LeetCode - 单词拆分](https://leetcode.cn/problems/word-break/description/?envType=study-plan-v2&envId=top-100-liked)**
>
> - 给你一个字符串 `s` 和一个字符串列表 `wordDict` 作为字典。如果可以利用字典中出现的一个或多个单词拼接出 `s` 则返回 `true`。注意不要求字典中出现的单词全部都使用，并且字典中的单词可以重复使用。
>
> ```JavaScript
> /**
>  * @param {string} s
>  * @param {string[]} wordDict
>  * @return {boolean}
>  */
> var wordBreak = function(s, wordDict) {
>   // 初始化 dp 表
>   // dp[i]表示字符串s的前i个字符能否由wordDict中的单词组成
>   const dp = Array(s.length + 1).fill(false)
> 
>   // 初始状态 => 空字符串总是可以被表示
>   dp[0] = true
> 
>   // 算法思想：
>   // 如果字符串s的前i个字符可以由字典中的单词组合而成
>   // 那么存在至少一个j（0 <= j < i），使得s的前j个字符可以由字典中的单词组合而成
>   // 并且j到i的这段子字符串同样是字典中的某个单词
> 
>   // 状态转移方程
>   for (let i = 1; i <= s.length; i++) {
>     // 遍历前 i 个字符，搜索是否存在一个j，使得s的前j个字符可以由字典中的单词组合而成
>     for (let j = 0; j < i; j++) {
>       if (dp[j] && wordDict.includes(s.substring(j, i))) {
>         dp[i] = true
>         break  // 找到一种分割方式即可
>       }
>     }
>   }
> 
>   return dp[s.length]
> };
> ```

# **03. 最长递增子序列**

> _**问题描述**_
>
> > [!important] **[LeetCode - 最长递增子序列](https://leetcode.cn/problems/longest-increasing-subsequence/description/?envType=study-plan-v2&envId=top-100-liked)**
>
> - 给你一个整数数组 `nums` ，找到其中最长严格递增子序列的长度。子序列是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如`[3,6,2,7]`是数组`[0,3,1,6,2,2,7]`的子序列。
>
> ```JavaScript
> var lengthOfLIS = function(nums) {
>   if (nums.length === 0) return 0
>   // 初始化 dp 表，dp[i] 表示 以第i个数字结尾的最长递增子序列的长度
>   // 每个元素自身可以构成长度为 1 的递增子序列，所以 dp 数组应该全部初始化为 1
>   const dp = Array(nums.length).fill(1)
>   let maxLen = 1
>   // 状态转移方程
>   // 要计算dp[i]，需要考虑所有小于i的j
>   for (let i = 1; i < dp.length; i++) {
>     for (let j = 0; j < i; j++) {
>       if (nums[i] > nums[j]) dp[i] = Math.max(dp[i], dp[j] + 1)
>     }
>     // 整个数组的最长递增子序列的长度等于 dp 数组中的最大值
>     // 因为最长递增子序列可以以数组中任何一个元素结尾
>     maxLen = Math.max(maxLen, dp[i])
>   }
>   return maxLen
> };
> ```

# **04. 乘积最大子数组**

> _**问题描述**_
>
> > [!important] **[LeetCode - 乘积最大子数组](https://leetcode.cn/problems/maximum-product-subarray/description/?envType=study-plan-v2&envId=top-100-liked)**
>
> - 给你一个整数数组 `nums` ，请你找出数组中乘积最大的非空连续子数组（该子数组中至少包含一个数字），并返回该子数组所对应的乘积。测试用例的答案是一个 **32-位** 整数。**子数组** 是数组的连续子序列。
>
> ```JavaScript
> // 关键在于考虑到乘积的特性，特别是负数的处理。
> // 一个负数乘以一个负数可以得到一个正数，这可能导致之前的最小乘积变成当前的最大乘积，反之亦然。
> // 因此，我们不仅需要跟踪到目前为止的最大乘积，还需要跟踪到目前为止的最小乘积，
> // 因为最小的负乘积通过乘以一个负数可以成为最大的正乘积。
> var maxProduct = function(nums) {
>   // 初始化 DP 表
>   // 创建两个数组maxProd和minProd，分别用于存储到当前位置为止的最大乘积和最小乘积。
>   // maxProd[i]表示到第i个元素为止的最大乘积子数组
>   // minProd[i]表示到第i个元素为止的最小乘积子数组
>   // 还需要一个变量result用于存储最终的最大乘积，初始时可以设置为nums[0]
>   const maxProd = Array(nums.length)
>   const minProd = Array(nums.length)
>   let result = nums[0]
> 
>   // 初始状态
>   maxProd[0] = nums[0]
>   minProd[0] = nums[0]
> 
>   // 状态转移方程
>   for (let i = 1; i < nums.length; i++) {
>     // 考虑到当前元素可能是正数也可能是负数，当前位置的最大值可以从以下三个值中产生
>     // 当前元素自身nums[i]。
>     // 当前元素与之前位置的最大乘积相乘maxProd[i-1] * nums[i]。
>     // 当前元素与之前位置的最小乘积相乘minProd[i-1] * nums[i]（特别是当nums[i]为负数时）。
>     // 同理，当前位置的最小值也需要计算，因为它可能在后续变为最大值
>     maxProd[i] = Math.max(nums[i], nums[i] * maxProd[i-1], nums[i] * minProd[i-1])
>     minProd[i] = Math.min(nums[i], nums[i] * maxProd[i-1], nums[i] * minProd[i-1])
>     // 更新全局最大乘积
>     result = Math.max(result, maxProd[i]);
>   }
> 
>   return result
> };
> ```

# 分割等和子集

> _**问题描述**_
>
> > [!important] **[LeetCode - 分割等和子集](https://leetcode.cn/problems/partition-equal-subset-sum/description/?envType=study-plan-v2&envId=top-100-liked)**
>
> - 给你一个 **只包含正整数** 的 **非空** 数组 `nums` 。请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

> _**0-1 背包问题的变种**_
>
> - 判断一个只包含正整数的非空数组是否可以被分割成两个元素和相等的子集，可以转换成一个求子集和问题
> 
> - 即是否存在一个子集，其和等于整个数组和的一半
> 
> - 如果数组的总和是奇数，则不可能分割成两个和相等的子集，因为和的一半不是整数。
> 
> - 如果数组的总和是偶数，问题就转化为了寻找是否存在一个子集，其和等于总和的一半

> _**第一步：思考每轮的决策，定义状态，从而得到 dp 表**_
>
> - 我们可以将问题转化为在数组中找到一些数字，使得它们的和等于数组总和的一半。
> 
> - 因此，我们可以定义状态`**dp[i][j]**`表示在前 _**i**_ 个数字中是否存在一些数字的和等于 _**j**_
> 
> - 如果存在，则`**dp[i][j]**`为 _**true**_，否则为 _**false**_

> _**第二步：找出最优子结构，进而推导出状态转移方程**_
>
> - 假设当前考虑的数字为 ==`**nums[i]**`==，我们有两种选择
> 
> - 不包括 `**nums[i]**`，则 `**dp[i][j] = dp[i-1][j]**`
> 
> - 包括 ==`**nums[i]**`==，则 ==`**dp[i][j] = dp[i-1][j-nums[i]]**`==
> 
> - 综合考虑以上两种情况，状态转移方程为
>     
>     - `**dp[i][j] = dp[i-1][j] || dp[i-1][j-nums[i]]**`
>     

> _**第三步：确定边界条件和状态转移顺序**_
>
> - 当 _**j**_ 等于 _**0**_ 时，`**dp[i][j]**`总是为 _**true**_，因为不选择任何数字即可得到和为 _**0**_ 的子集
> 
> - 当 _**i**_ 等于 _**0**_ 时，`**dp[i][j]**`只有在 _**j**_ 等于 `**nums[0]**` 时才为 _**true**_，其他情况都为 _**false**_
> 
> - 由于`**dp[i][j]**`的计算依赖于`**dp[i-1][j]**`和`**dp[i-1][j-nums[i]]**`，因此需要先计算前面的状态值。

> _**代码详解**_
>
> ```JavaScript
> var canPartition = function(nums) {
>   const n = nums.length;
>   // 先求解数组元素的总和
>   const sumTotal = nums.reduce((prev, cur) => prev + cur, 0)
>   // 判断 sumTotal 的奇偶性
>   if (sumTotal % 2 === 1) return false
>   const target = sumTotal / 2
>   const dp = Array.from({ length: n + 1 }, () => Array(target + 1).fill(false));
>   // 当 j 等于 0 时，dp[i][j]总是为 true，因为不选择任何数字即可得到和为 0 的子集
>   for (let row = 0; row < dp.length; row++) dp[row][0] = true
>   for (let i = 1; i <= n; i++) {
>       for (let j = 1; j <= target; j++) {
>           if (j >= nums[i - 1]) {
>               dp[i][j] = dp[i - 1][j] || dp[i - 1][j - nums[i - 1]];
>           } else {
>               dp[i][j] = dp[i - 1][j];
>           }
>       }
>   }
>   return dp[n][target];
> };
> ```

### _**最长回文子串**_

> _**问题描述**_
>
> > [!important] **[LeetCode - 最长回文子串](https://leetcode.cn/problems/longest-palindromic-substring/description/?envType=study-plan-v2&envId=top-100-liked)**
>
> - 给你一个字符串 `s`，找到 `s` 中最长的回文子串。如果字符串的反序与原始字符串相同，则该字符串称为回文字符串。

> _**第一步：思考每轮的决策，定义状态，从而得到 dp 表**_
>
> - 定义 `**dp[i][j]**` 为布尔值，表示字符串中从第 `**i**` 个字符到第 `**j**` 个字符组成的子串是否是回文子串
> 
> - _==**只会使用上三角矩阵**==_
> 
> - 如果是回文子串，`**dp[i][j]**` 为 `**true**`；否则为 `**false**`

> _**第二步：找出最优子结构，进而推导出状态转移方程**_
>
> - 对于任意子串 `**s[i...j]**`，如果 `**s[i] === s[j]**` 并且 `**s[i+1...j-1]**` 也是回文串（即 `**dp[i+1][j-1] == true**`），那么 `**s[i...j]**` 是回文串，即 `**dp[i][j] = true**`
> 
> - 具体转移方程如下：_==`**dp[i][j] = ( s[i] === s[j] ) && ( j−i<3 || dp[i+1][j−1] )**`==_
> 
> - 这里的 `**j - i < 3**` 是为了处理边界条件，即当子串长度为 _**2**_ 或 _**3**_ 时，只需要检查 `**s[i]**` 和 `**s[j]**` 是否相等

> _**第三步：确定边界条件和状态转移顺序**_
>
> - **边界条件**：对于任何 `**i**`，`**dp[i][i]**` 一定是 `**true**`，因为单个字符总是回文串
>     
>     - 此外，初始化时所有 `**dp[i][j]**` 应为 `**false**`
>     
> 
> - **状态转移顺序**：由于 `**dp[i][j]**` 的计算依赖于 `**dp[i+1][j-1]**`，我们需要从下往上，从左往右遍历 `**dp**` 表
>     
>     - 可以先计算所有 `**dp[i][i]**`（单个字符），然后是所有两个字符的子串，依此类推
>     

> _**代码详解**_
>
> ```JavaScript
> var longestPalindrome = function(s) {
>   const n = s.length
> 
>   // 用两个变量记录最长回文串的起始点和长度
>   // 默认最长回文子串为第一个字符
>   let start = 0, maxLen = 1
> 
>   // 创建 dp 表并初始化
>   // dp[i][j]表示字符串中从第 i 个字符到第 j 个字符组成的子串是否是回文子串
>   // 只会使用上三角矩阵
>   const dp = Array.from({ length: n }, () => Array(n).fill(false))
> 
>   // 初始化状态 / 处理边界 1 => 处理单个字符的子串
>   // 单个字符都是回文
>   for (let i = 0; i < n; i++) dp[i][i] = true
> 
>   // 初始化状态 / 处理边界 2 => 处理两个字符的子串
>   for (let i = 0; i < n - 1; i++) {
>     if (s[i] === s[i+1]) {
>       dp[i][i+1] = true
>       // 更新最长回文串的起始点和长度
>       start = i
>       maxLen = 2
>     }
>   }
> 
>   // 处理长度大于 2 的子串
>   for (let len = 3; len <= n; len++) {
>     // 注意 i 的取值范围，根据 len 的长度改变 => i 表示子串的起始点
>     for (let i = 0; i < n - len + 1; i++) {
>       // j 表示子串的结束点
>       let j = i + len - 1 
>       
>       // 对于任意子串 s[i...j]，如果 s[i] === s[j] 
>       // 并且 s[i+1...j-1] 也是回文串（即 dp[i+1][j-1] == true）
>       // 那么 s[i...j] 是回文串，即 dp[i][j] = true
>       if (s[i] === s[j] && dp[i+1][j-1]) {
>         dp[i][j] = true
>         // 更新最长回文串的起始点和长度
>         start = i
>         maxLen = len
>       }
>     }
>   }
> 
>   return s.substring(start, start+maxLen)
> };
> ```

### _**最长公共子序列**_

> _**问题描述**_
>
> > [!important] **[LeetCode - 最长公共子序列](https://leetcode.cn/problems/longest-common-subsequence/description/?envType=study-plan-v2&envId=top-100-liked)**
>
> - 给定两个字符串 `text1` 和 `text2`，返回这两个字符串的最长 **公共子序列** 的长度。如果不存在 **公共子序列** ，返回 `0` 。一个字符串的 **子序列** 是指这样一个新的字符串：它是由原字符串在不改变字符的相对顺序的情况下删除某些字符（也可以不删除任何字符）后组成的新字符串。例如，`"ace"` 是 `"abcde"` 的子序列，但 `"aec"` 不是 `"abcde"` 的子序列。两个字符串的 **公共子序列** 是这两个字符串所共同拥有的子序列。

> _**第一步：思考每轮的决策，定义状态，从而得到 dp 表**_
>
> - 定义状态`**dp[i][j]**`表示 _**text1**_ 的前 _**i**_ 个字符和 _**text2**_ 的前 _**j**_ 个字符的最长公共子序列长度

> _**第二步：找出最优子结构，进而推导出状态转移方程**_
>
> 假设当前考虑的字符分别是 `**text1[i-1]**` 和 `**text2[j-1]**`：
>
> 1. 如果`**text1[i-1]**`等于`**text2[j-1]**`，那么它们一定属于最长公共子序列中
>     
>     - 此时最长公共子序列长度为 ==`**dp[i-1][j-1] + 1**`==
>     
> 
> 2. 如果`**text1[i-1]**`不等于`**text2[j-1]**`，那么它们不能同时出现在最长公共子序列中
>     
>     - 此时最长公共子序列长度为 ==`**max(dp[i-1][j], dp[i][j-1])**`==
>     
> 
> - 综合考虑以上两种情况，得到状态转移方程：
> 
> - `**dp[i][j] = dp[i-1][j-1] + 1, if text1[i-1] == text2[j-1]**`
> 
> - `**dp[i][j] = max(dp[i-1][j], dp[i][j-1]), if text1[i-1] != text2[j-1]**`

> _**第三步：确定边界条件和状态转移顺序**_
>
> - 当 _**i**_ 或 _**j**_ 等于 _**0**_ 时，`**dp[i][j]**`必定为 _**0**_，因为其中一个字符串为空字符串，它们之间不存在公共子序列
> 
> - 状态转移顺序
>     
>     - 由于`**dp[i][j]**`的计算依赖于`**dp[i-1][j-1]、dp[i-1][j] 和 dp[i][j-1]**`，因此需要先计算前面的状态值。
>     

> _**代码详解**_
>
> ```JavaScript
> var longestCommonSubsequence = function(text1, text2) {
>   const n = text1.length + 1, m = text2.length + 1
> 
>   // 初始化 dp 表
>   // dp[i] 表示 表示 text1 的前 i 个字符和 text2 的前 j 个字符的最长公共子序列长度
>   const dp = Array.from({ length: n }, () => Array(m).fill(0))
> 
>   // 状态转移方程
>   for (let i = 1; i < n; i++) {
>     for (let j = 1; j < m; j++) {
>       // 如果text1[i-1]等于text2[j-1]，那么它们一定属于最长公共子序列中
>       if (text1[i-1] === text2[j-1]) dp[i][j] = dp[i-1][j-1] + 1
>       // 如果text1[i-1]不等于text2[j-1]，那么它们不能同时出现在最长公共子序列中
>       else dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1])
>     }
>   }
> 
>   return dp[n-1][m-1]
> };
> ```

# _**爬楼梯**_

> _**问题描述**_
>
> - 给定一个共有 **n** 阶的楼梯，你每步可以上 **1** 阶或者 **2** 阶。请问有多少种方案可以爬到楼顶？
> 
> ![[动态规划 - 13.png|Untitled 86.png|357]]

> _**使用动态规划**_
>
> - _==**动态规划是一种 “从底至顶 ” 的方法。**==_从最小子问题的解开始，迭代地构建更大子问题的解，直至得到原问题的解。由于动态规划不包含回溯过程，因此只需使用循环迭代实现，无须使用递归。
> 
> ```JavaScript
> /* 爬楼梯：动态规划 */
> function climbingStairsDP(n) {
>     if (n === 1 || n === 2) return n;
>     // 初始化 dp 表，用于存储子问题的解
>     const dp = new Array(n + 1).fill(-1);
>     // 初始状态：预设最小子问题的解
>     dp[1] = 1;
>     dp[2] = 2;
>     // 状态转移：从较小子问题逐步求解较大子问题
>     for (let i = 3; i <= n; i++) {
>         dp[i] = dp[i - 1] + dp[i - 2];
>     }
>     return dp[n];
> }
> ```

### _**0-1 背包问题**_

> _**Reference**_
>
> > [!important] **[Hello 算法 - 0-1 背包问题](https://www.hello-algo.com/chapter_dynamic_programming/knapsack_problem/)**

> _**问题描述**_
>
> - 给定 **n** 个物品，第 **i** 个物品的重量为`**wgt[i-1]**`、价值为`**val[i-1]**`，和一个容量为 **cap** 的背包。由于物品编号从 **1** 开始计数，数组索引从 **0** 开始计数，因此物品 **i** 对应重量`**wgt[i-1]**`和价值`**val[i-1]**`**。**每个物品 只能选择一次，问在限定背包容量下能放入物品的最大价值。
> 
> ![[动态规划 - 14.png|Untitled 1 57.png|260]]

> _**第一步：思考每轮的决策，定义状态，从而得到 dp 表**_
>
> - 每一轮的决策是 是否将物品放入背包。对于每个物品来说，不放入背包，背包容量不变；放入背包，背包容量减小
> 
> - 由此可得状态定义 ⇒ 当前物品编号 **i** 和剩余背包容量 **c**，记为`**[i, c]**`
> 
> - `**[i, c]**`对应的子问题 ⇒ 前 **i** 个物品在剩余容量为 **c** 的背包中的最大价值，记为`**dp[i,c]**`
> 
> - 待求解的是`**dp[n,cap]**`，因此需要一个尺寸为==`**(n + 1)×(cap + 1)**`==的二维 **dp** 表

> _**第二步：找出最优子结构，进而推导出状态转移方程**_
>
> - 当我们做出物品 _**i**_ 的决策时，可分为以下两种情况
> 
> - _**情况 1 ⇒ 不放入物品 i ⇒**_ 背包容量不变，最大价值不变，状态变化为`**[i-1,c]**`；前 **i** 个物品的最大价值 等于 前 **i-1** 个物品的最大价值，即`**dp[i,c] = dp[i-1,c]**`
> 
> - _**情况 2 ⇒ 放入物品 i ⇒**_ 背包剩余容量 **c** 减少==`**wgt[i-1]**`====，价值增加====`**val[i-1]**`====，状态变化为====`**[i-1, c-wgt[i-1]]**`====**。**==前 **i** 个物品的最大价值 = 前 **i-1** 个物品==**在**====`**c-wgt[i-1]**`====**容量下**==的最大价值 + 物品 **i** 的价值，即`**dp[i,c] = dp[i-1, c - wgt[i-1]] + val[i-1]**`
> 
> - _**特殊情况**_ ⇒ 若当前物品重量==`**wgt[i-1]**`==超出剩余背包容量 _**c**_ ，则只能选择不放入背包
> 
> - _**最优子结构 ⇒**_ 最大价值`**dp[i,c]**`等于不放入物品 _**i**_ 和放入物品 _**i**_ 两种方案中价值更大的一个
> 
> $$dp[i,c] = max(dp[i-1,c], dp[i-1,c-wgt[i-1]]+val[i-1])$$

> _**第三步：确定边界条件和状态转移顺序**_
>
> - 当 无物品 或 无剩余背包容量 时最大价值为 _**0**_，即首列`**dp[i,0]**`和首行`**dp[0,c]**`都等于 _**0**_
> 
> - 当前状态`**[i,c]**`从上方的状态`**[i-1,c]**`和左上方的状态`**[i-1,c - wgt[i-1]]**`转移而来
> 
> - 因此通过两层循环正序遍历整个 _**dp**_ 表即可

> _**图解 & 代码详解**_
>
> > [!important] **[Hello 算法 - 完整图解](https://www.hello-algo.com/chapter_dynamic_programming/knapsack_problem/#3)**
>
> ![[动态规划 - 15.png|Untitled 2 53.png]]
>
> ![[动态规划 - 16.png|Untitled 3 40.png]]
>
> ```JavaScript
> /* 0-1 背包：动态规划 */
> function knapsackDP(wgt, val, cap) {
>   const n = wgt.length;
>   // 初始化 dp 表
>   const dp = Array(n + 1).fill(0).map(() => Array(cap + 1).fill(0));
>   // 状态转移
>   for (let i = 1; i <= n; i++) {
>     for (let c = 1; c <= cap; c++) {
>       if (wgt[i - 1] > c) {
>         // 若超过背包容量，则不选物品 i
>         dp[i][c] = dp[i - 1][c];
>       } else {
>         // 不选和选物品 i 这两种方案的较大值
>         dp[i][c] = Math.max(dp[i - 1][c], dp[i - 1][c - wgt[i - 1]] + val[i - 1]);
>       }
>     }
>   }
>   return dp[n][cap];
> }
> 
> // 空间优化
> // 由于每个状态都只与其上一行的状态有关，可以只使用一个数组存储二维数组中列的信息
> function knapsackDPComp(wgt, val, cap) {
>     const n = wgt.length;
>     // 初始化 dp 表
>     const dp = Array(cap + 1).fill(0);
>     // 状态转移
>     for (let i = 1; i <= n; i++) {
>         // 倒序遍历，不会发生覆盖问题
>         for (let c = cap; c >= 1; c--) {
>             if (wgt[i - 1] <= c) {
>                 // 不选和选物品 i 这两种方案的较大值
>                 dp[c] = Math.max(dp[c], dp[c - wgt[i - 1]] + val[i - 1]);
>             }
>         }
>     }
>     return dp[cap];
> }
> ```

### _**打家劫舍**_

> _**Reference**_
>
> > [!important] **[LeetCode - 打家劫舍](https://leetcode.cn/problems/house-robber/description/?envType=study-plan-v2&envId=top-100-liked)**

> _**问题描述**_
>
> - 你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，**如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。**给定一个代表每个房屋存放金额的非负整数数组，计算你 **不触动警报装置的情况下** ，一夜之内能够偷窃到的最高金额。

> _**第一步：思考每轮的决策，定义状态，从而得到 dp 表**_
>
> - _**每一轮的决策 ⇒**_ 是否偷窃第 **i** 个房屋
> 
> - _**由此可得状态定义 ⇒**_ 当前房屋编号 **i**
> 
> - _**i 对应的子问题 ⇒**_ 前 **i** 个房屋能偷窃到的最大金额，记为`**dp[i]**`
> 
> - 待求解的是`**dp[n]**`，因此需要一个尺寸为 **n** 的一维 **dp** 表

> _**第二步：找出最优子结构，进而推导出状态转移方程**_
>
> - 当做出房屋 **i** 的决策时，可分为以下两种情况
> 
> - _**情况 1 ⇒ 不偷窃房屋 i ⇒**_ 状态变化为 **i-1，**前 **i** 个房屋的最大金额等于前 **i-1** 个房屋的最大金额，即`**dp[i] = dp[i-1]**`
> 
> - _**情况 2 ⇒ 偷窃房屋 i ⇒**_ ==状态变化为== ==**i-2**==，因为不能偷相邻的房屋，所以前 **i** 个房屋的最大金额 = 前 **i-2** 个房屋的最大金额 + 第 **i** 个房屋的金额，即`**dp[i] = dp[i-2] + nums[i]**`
> 
> - _**最优子结构 ⇒**_ 最大金额`**dp[i]**`等于 不偷窃房屋 **i** 和 偷窃房屋 **i** 两种方案中价值更大的一个
> 
> $$dp[i] = max(dp[i-1], dp[i-2]+nums[i])$$

> _**第三步：确定边界条件和状态转移顺序**_

> _**代码详解**_
>
> ```JavaScript
> /**
>  * @param {number[]} nums
>  * @return {number}
>  */
> var rob = function(nums) {
>   if (nums.length === 0) return 0
>   if (nums.length === 1) return nums[0]
> 
>   // 1. 初始化 dp 表
>   const dp = Array(nums.length)
> 
>   // 2. 初始状态
>   dp[0] = nums[0]
>   dp[1] = Math.max(nums[0], nums[1])
> 
>   // 3. 状态转移方程
>   for (let i = 2; i < nums.length; i++) {
>     dp[i] = Math.max(dp[i-1], dp[i-2]+nums[i])
>   }
> 
>   return dp[nums.length - 1]
> };
> ```

### _**完全背包问题**_

> _**Reference**_
>
> > [!important] **[Hello 算法 - 完全背包问题](https://www.hello-algo.com/chapter_dynamic_programming/unbounded_knapsack_problem/)**

> _**问题描述**_
>
> - 给定 **n** 个物品，第 **i** 个物品的重量为`**wgt[i-1]**`、价值为`**val[i-1]**`，和一个容量为 **cap** 的背包。由于物品编号从 **1** 开始计数，数组索引从 **0** 开始计数，因此物品 **i** 对应重量`**wgt[i-1]**`和价值`**val[i-1]**`。每个物品可以重复选取，问在限定背包容量下能放入物品的最大价值。
> 
> ![[动态规划 - 18.png|Untitled 5 25.png|278]]

> _**完全背包问题和 0-1 背包问题非常相似，区别仅在于****不限制物品的选择次数**_
>
> - _**0-1**_ 背包问题中，每种物品只有一个，因此将物品放入背包后，只能从前 _**i-1**_ 个物品中选择
> 
> - 完全背包问题中，每种物品数量是无限的，将物品放入背包后，仍可以从前 _**i**_ 个物品中选择
> 
> - _**在完全背包问题的规定下，状态 [i, c] 的变化分为两种情况**_
>     
>     - _**不放入物品 i ⇒**_ 与 _**0-1**_ 背包问题相同，转移至`**[i-1,c]**`
>     
>     - _**放入物品 i ⇒**_ 与 _**0-1**_ 背包问题不同，转移至`**[i,c - wgt[i-1]]**`
>     
>     $$dp[i,c] = max(dp[i-1,c], dp[i,c-wgt[i-1]]+val[i-1])$$
>     
> 
> ```JavaScript
> /* 完全背包：动态规划 */
> function unboundedKnapsackDP(wgt, val, cap) {
>     const n = wgt.length;
>     // 初始化 dp 表
>     const dp = Array.from({ length: n + 1 }, () =>
>         Array.from({ length: cap + 1 }, () => 0)
>     );
>     // 状态转移
>     for (let i = 1; i <= n; i++) {
>         for (let c = 1; c <= cap; c++) {
>             if (wgt[i - 1] > c) {
>                 // 若超过背包容量，则不选物品 i
>                 dp[i][c] = dp[i - 1][c];
>             } else {
>                 // 不选和选物品 i 这两种方案的较大值
>                 dp[i][c] = Math.max(
>                     dp[i - 1][c],
>                     dp[i][c - wgt[i - 1]] + val[i - 1]
>                 );
>             }
>         }
>     }
>     return dp[n][cap];
> }
> ```

### _**零钱兑换问题**_

> _**Reference**_
>
> > [!important] **[Hello 算法 - 零钱兑换问题](https://www.hello-algo.com/chapter_dynamic_programming/unbounded_knapsack_problem/#3)**

> _**问题描述**_
>
> - 给定 _**n**_ 种硬币，第 _**i**_ 种硬币的面值为`**coins[i-1]**`，目标金额为 _**amt**_
> 
> - 每种硬币可以 重复选取，问能够凑出目标金额的最少硬币数量
> 
> - 如果无法凑出目标金额，则返回 _**-1**_
> 
> ![[动态规划 - 19.png|Untitled 6 20.png|289]]

> _**零钱兑换 可以看作 完全背包问题 的一种特殊情况**_
>
> - 两道题可以相互转换：物品 ⇒ 硬币；物品重量 ⇒ 硬币面值；背包容量 ⇒ 目标金额
> 
> - 优化目标相反，完全背包问题是要最大化物品价值，零钱兑换问题是要最小化硬币数量
> 
> - 完全背包问题是求“不超过”背包容量下的解，零钱兑换是求“恰好”凑到目标金额的解

> _**第一步：思考每轮的决策，定义状态，从而得到 dp 表**_
>
> - 状态`**[i,a]**`对应的子问题：前 _**i**_ 种硬币能够凑出金额 _**a**_ 的最少硬币数量，记为`**dp[i,a]**`
> 
> - 二维 _**dp**_ 表的尺寸为==`**(n + 1) × (amt + 1)**`==

> _**第二步：找出最优子结构，进而推导出状态转移方程**_
>
> - 本题与完全背包问题的状态转移方程存在以下两点差异
> 
> - 本题要求最小值，因此需将运算符`**max()**`更改为`**min()**`
> 
> - 优化主体是 硬币数量而非商品价值，因此在选中硬币时执行 _**+1**_ 即可
> 
> $$dp[i,a]=min(dp[i-1,a],dp[i,a-coins[i-1]]+1)$$

> **第三步：确定边界条件和状态转移顺序**
>
> - 当目标金额为 _**0**_ 时，凑出它的最少硬币数量为 0，即首列所有`**dp[i,0]**`都等于 _**0**_
> 
> - _==**当 无硬币 时，**====**无法凑出任意 > 0 的目标金额**====**，即是无效解**==_
>     
>     - 为使状态转移方程中的 _**min**_ 函数能够 识别并过滤无效解
>     
>     - 使用 _**+∞**_ 来表示它们，即令首行所有`**dp[0,a]**`都等于 _**+∞**_
>     
>     - 采用数字 _**amt+1**_ 来表示无效解，因为凑出 _**amt**_ 的硬币数量最多为 _**amt（全为硬币 1）**_
>     
>     - 判断==`**dp[n,amt]**`==是否等于 _**amt+1**_，若是则返回 _**-1**_，代表无法凑出目标金额
>     

> _**Code**_
>
> ```JavaScript
> /* 零钱兑换：动态规划 */
> function coinChangeDP(coins, amt) {
>     const n = coins.length;
>     const MAX = amt + 1;
>     // 初始化 dp 表
>     const dp = Array.from({ length: n + 1 }, () =>
>         Array.from({ length: amt + 1 }, () => 0)
>     );
>     // 状态转移：首行首列
>     for (let a = 1; a <= amt; a++) {
>         dp[0][a] = MAX;
>     }
>     // 状态转移：其余行和列
>     for (let i = 1; i <= n; i++) {
>         for (let a = 1; a <= amt; a++) {
>             if (coins[i - 1] > a) {
>                 // 若超过目标金额，则不选硬币 i
>                 dp[i][a] = dp[i - 1][a];
>             } else {
>                 // 不选和选硬币 i 这两种方案的较小值
>                 dp[i][a] = Math.min(dp[i - 1][a], dp[i][a - coins[i - 1]] + 1);
>             }
>         }
>     }
>     return dp[n][amt] !== MAX ? dp[n][amt] : -1;
> }
> ```

### _**零钱兑换问题 II**_

> _**问题描述**_
>
> - 给定 _**n**_ 种硬币，第 _**i**_ 种硬币的面值为`**coins[i-1]**`，目标金额为 _**amt**_
> 
> - 每种硬币 可以重复选取，问凑出目标金额的硬币组合数量
> 
> ![[动态规划 - 20.png|Untitled 7 15.png|275]]

> _**相比于上一题，本题目标是求组合数量**_
>
> - 子问题变为 ⇒ 前 _**i**_ 种硬币能够凑出金额 _**a**_ 的组合数量
> 
> - 而 _**dp**_ 表仍然是尺寸为`**(n+1) × (amt+1)**`的二维矩阵
> 
> - 当前状态的组合数量 等于 不选当前硬币 与 选当前硬币 这两种决策的组合数量之和
> 
> $$dp[i,a]=dp[i-1,a]+dp[i,a-coins[i-1]]$$
>
> - 当目标金额为 _**0**_ 时，无须选择任何硬币即可凑出目标金额，因此应将首列所有`**dp[i,0]**`都初始化为 _**1**_
> 
> - 当 无硬币 时，无法凑出任何 > 0 的目标金额，因此首行所有`**dp[0,a]**`都等于 _**0**_

> _**Code**_
>
> ```JavaScript
> /* 零钱兑换 II：动态规划 */
> function coinChangeIIDP(coins, amt) {
>     const n = coins.length;
>     // 初始化 dp 表
>     const dp = Array.from({ length: n + 1 }, () =>
>         Array.from({ length: amt + 1 }, () => 0)
>     );
>     // 初始化首列
>     for (let i = 0; i <= n; i++) {
>         dp[i][0] = 1;
>     }
>     // 状态转移
>     for (let i = 1; i <= n; i++) {
>         for (let a = 1; a <= amt; a++) {
>             if (coins[i - 1] > a) {
>                 // 若超过目标金额，则不选硬币 i
>                 dp[i][a] = dp[i - 1][a];
>             } else {
>                 // 不选和选硬币 i 这两种方案之和
>                 dp[i][a] = dp[i - 1][a] + dp[i][a - coins[i - 1]];
>             }
>         }
>     }
>     return dp[n][amt];
> }
> ```

# _**动态规划理解 &** DP 问题判断_

> _**动态规划理解 & 子问题分解**_
>
> - 动态规划 **dynamic programming** 是一个重要的算法范式。它将一个问题分解为一系列更小的子问题，并通过存储子问题的解来避免重复计算，从而大幅提升时间效率。动态规划常用来求解最优化问题。
> 
> - 子问题分解是一种通用的算法思路，在分治、动态规划、回溯中的侧重点不同
> 
> - 分治算法递归地将原问题划分为 多个相互独立的子问题，直至最小子问题，并在回溯中合并子问题的解，最终得到原问题的解。
> 
> - _==**与 分治算法 的主要区别是，**====**动态规划中的 子问题 是 相互依赖 的**====**，在分解过程中**== ==**会出现许多重叠子问题**==_
> 
> - 回溯算法在尝试和回退中穷举所有可能的解，并通过剪枝避免不必要的搜索分支。原问题的解由一系列决策步骤构成，我们可以将每个决策步骤之前的子序列看作一个子问题

> _**如何判断一个题目是否是 DP 问题？**_
>
> - _**先观察问题是否适合使用 回溯 解决 ⇒**_ 适合用回溯解决的问题通常满足“决策树模型”，这种问题可以使用树形结构来描述。其中每一个节点代表一个决策，每一条路径代表一个决策序列。如果问题包含明确的决策概念，并且解是通过一系列决策产生的，那么它就满足决策树模型，通常可以使用回溯来解决
> 
> - _==**DP 问题的特点**== **⇒ 1.**_ 问题包含最大（小）或最多（少）等最优化描述。_**2.**_ 问题的状态能够使用一个列表、多维矩阵或树来表示。_**3.**_ 一个状态与其周围的状态存在递推关系

> _**动态规划的常用术语**_
>
> - 将数组 **dp** 称为 **dp** 表，`**dp[i]**`表示状态 **i** 对应子问题的解
> 
> - _**初始状态 ⇒**_ 最小子问题对应的状态（第 **1** 阶和第 **2** 阶楼梯）
> 
> - _**状态转移方程 ⇒**_ 递推公式`**dp[i] = dp[i-1] + dp[i-2]**`，从较小子问题逐步求解较大子问题
>

### _**最优子结构 & 爬楼梯最小代价**_

> _**最优子结构 & 爬楼梯最小代价**_
>
> - _**最优子结构 ⇒**_ 原问题的最优解是从子问题的最优解构建得来的
> 
> - _**问题描述 ⇒**_ 给定一个楼梯，你每步可以上 **1** 阶或者 **2** 阶。每一阶楼梯上都贴有一个非负整数，表示你在该台阶所需要付出的代价。给定一个非负整数数组 **cost**，其中`**cost[i]**`表示在第 **i** 个台阶需要付出的代价。`**cost[0]**`为地面（起始点）。请计算最少需要付出多少代价才能到达顶部？
> 
> - _**算法分析 ⇒**_ 设 `dp[i]` 为爬到第 **i** 阶累计付出的代价。由于第 **i** 阶只可能从 **i - 1** 阶或 **i - 2** 阶走来，因此 `dp[i]` 只可能等于 `dp[i-1] + cost[i]` 或 `dp[i-2] + cost[i]`。为了尽可能减少代价，我们应该选择两者中较小的那一个，即 `**dp[i] = min(dp[i-1],dp[i-2]) + cost[i]`
> 
> ![[动态规划 - 7.png|Untitled 1 56.png]]
>
> ```JavaScript
> /* 爬楼梯最小代价：动态规划 */
> function minCostClimbingStairsDP(cost) {
>     const n = cost.length - 1;
>     if (n === 1 || n === 2) {
>         return cost[n];
>     }
>     // 初始化 dp 表，用于存储子问题的解
>     const dp = new Array(n + 1);
>     // 初始状态：预设最小子问题的解
>     dp[1] = cost[1];
>     dp[2] = cost[2];
>     // 状态转移：从较小子问题逐步求解较大子问题
>     for (let i = 3; i <= n; i++) {
>         dp[i] = Math.min(dp[i - 1], dp[i - 2]) + cost[i];
>     }
>     return dp[n];
> }
> ```

### _**无后效性 & 带约束的爬楼梯**_

> _**无后效性 & 带约束的爬楼梯**_
>
> - _**无后效性 ⇒**_ ==给定一个确定的状态，它的未来发展====只与当前状态有关，而与过去经历的所有状态无关====。==以爬楼梯为例，状态 **i** 会衍生出状态 **i+1** 和状态 **i+2**，分别对应跳 **1** 步和跳 **2** 步。在做出这两种选择时，无须考虑状态之前的状态，因为它们对未来的状态没有影响。
> 
> - _**问题描述 ⇒**_ 给定一个共有 **n** 阶的楼梯，你每步可以上 **1** 阶或者 **2** 阶，但不能连续两轮跳 **1** 阶。请问有多少种方案可以爬到楼顶？
> 
> - _**算法分析**_
>     
>     - 如果上一轮是跳 **1** 阶上来的，那么下一轮就必须跳 **2** 阶。因此，下一步选择不能由当前状态（当前所在楼梯阶数）独立决定，还和前一个状态（上一轮所在楼梯阶数）有关。
>     
>     - 如果使用一维 **dp** 数组，则此问题已不满足无后效性。因为`**dp[i-1]**`代表本轮跳 **1** 阶，但其中包含了许多“上一轮是跳 **1** 阶上来的”方案。为了满足约束，我们就不能将`**dp[i-1]**`直接计入`**dp[i]**`中。因此要使用二维 **dp** 数组实现无后效性。
>     
> 
> ![[动态规划 - 8.png|Untitled 2 52.png]]

> _**使用二维 dp 数组实现无后效性**_
>
> - 状态==`**[i,j]**`==表示处在第 **i** 阶并且上一轮跳了 **j** 阶，其中==`**j∈{1,2}**`==
> 
> - 此状态定义有效区分了上一轮跳了 **1** 阶还是 **2** 阶，可以据此判断当前状态是从何而来的
> 
> - 当上一轮跳了 **1** 阶时，上上一轮只能选择跳 **2** 阶。即`**dp[i,1]**`只能从`**dp[i-1, 2]**`转移过来
> 
> - 当上一轮跳了 **2** 阶时，上上一轮可选择跳 **1** 阶或跳 **2** 阶，即==`**dp[i,2]**`==可以从==`**dp[i-2,1]**`==或==`**dp[i-2,2]**`==转移过来。
> 
> $$dp[i,j]  
> \begin{cases} dp[i,1]=dp[i-1,2]  
> \\  
> dp[i,2]=dp[i-2,1]+dp[i-2,2]  
> \end{cases}$$
> 
> - 最终返回`**dp[n,1] + dp[n,2]**`即可，两者之和代表爬到第 **n** 阶的方案总数
> 
> ```JavaScript
> /* 带约束爬楼梯：动态规划 */
> function climbingStairsConstraintDP(n) {
>   if (n === 1 || n === 2) {
>     return 1;
>   }
>   // 初始化 dp 表(二维数组)，用于存储子问题的解
> 	// 相当于 const dp = Array.from(new Array(n + 1)).map(() => new Array(3))
> 	// Array.from(new Array(n + 1)) 会生成一个长度为 n+1 的数组，每个元素为 undefined
>   const dp = Array.from(new Array(n + 1), () => new Array(3));
>   // 初始状态：预设最小子问题的解
>   dp[1][1] = 1;
>   dp[1][2] = 0;
>   dp[2][1] = 0; // 不能连续跳 1 步，所以为 0
>   dp[2][2] = 1;
>   // 状态转移：从较小子问题逐步求解较大子问题
>   for (let i = 3; i <= n; i++) {
>     dp[i][1] = dp[i - 1][2];
>     dp[i][2] = dp[i - 2][1] + dp[i - 2][2];
>   }
>   return dp[n][1] + dp[n][2];
> }
> ```

### _DP 问题求解框架 & 最小路径和_

> _**通常遵循以下步骤**_
>
> - 描述决策，定义状态，建立 **dp** 表，推导状态转移方程，确定边界条件 等

> _**求解步骤示例 —— 最小路径和**_
>
> - 给定一个 **n×m** 的二维网格 **grid。**网格中的每个单元格包含一个非负整数，表示该单元格的代价。机器人以左上角单元格为起始点，每次只能向下或向右移动一步，直至到达右下角单元格。请返回从左上角到右下角的最小路径和。
> 
> ![[动态规划 - 9.png|Untitled 3 39.png]]

> _**第一步：思考每轮的决策，定义状态，从而得到 dp 表**_
>
> - 每一轮的决策就是从当前格子 向下或向右走一步
> 
> - 设当前格子的行列索引为`**[i,j]**`，则向下或向右走一步后，索引变为==`**[i+1,j]**`==或==`**[i,j+1]**`==
> 
> - 因此，状态应包含行索引和列索引两个变量，记为`**[i,j]**`
> 
> - 状态`**[i,j]**`对应的子问题为：从起始点`**[0,0]**`走到`**[i,j]**`的最小路径和，解记为==`**dp[i,j]**`==
> 
> - _==**从本质上看，dp 表是状态和子问题的解之间的映射**==_
> 
> ![[动态规划 - 10.png|Untitled 4 31.png]]

> _**第二步：找出最优子结构，进而推导出状态转移方程**_
>
> - 对于状态`**[i,j]**`，它只能从上边格子`**[i-1,j]**`和左边格子`**[i,j-1]**`转移而来
> 
> - 因此最优子结构为：到达==`**[i,j]**`==的最小路径和由==`**[i,j-1]**`==的最小路径和与==`**[i-1,j]**`==的最小路径和中较小的那一个决定
> 
> - 根据以上分析，可推出状态转移方程`**dp[i,j] = min(dp[i-1,j], dp[i,j-1]) + grid[i,j]**`
> 
> ![[动态规划 - 11.png|Untitled 5 24.png]]

> _**第三步：确定 边界条件 和 状态转移顺序**_
>
> - 在本题中，处在首行的状态只能从其左边的状态得来，处在首列的状态只能从其上边的状态得来，因此首行`**i=0**`和首列`**j=0**`是边界条件
> 
> - 由于每个格子是由其左方格子和上方格子转移而来，因此我们使用循环来遍历矩阵，外循环遍历各行，内循环遍历各列
> 
> ![[动态规划 - 12.png|Untitled 6 19.png]]

> _**代码实现**_
>
> ```JavaScript
> var minPathSum = function(grid) {
>   // 获取 grid 的行数和列数
>   const rows = grid.length, cols = grid[0].length
> 
>   // 初始化 dp 表，将所有元素初始化为 0
>   // const dp = Array.from({ length: rows }, () => Array.from({ length: cols }, () => 0))
>   const dp = Array.from({ length: rows }, () => Array(cols).fill(0))
> 
>   // 初始状态 1：[0, 0]元素
>   dp[0][0] = grid[0][0]
>   // 初始状态 2：首行
>   for (let i = 1; i < cols; i++) dp[0][i] = dp[0][i - 1] + grid[0][i]
>   // 初始状态 3：首列
>   for (let i = 1; i < rows; i++) dp[i][0] = dp[i - 1][0] + grid[i][0]
> 
>   // 状态转移方程
>   for (let i = 1; i < rows; i++) {
>     for (let j = 1; j < cols; j++) {
>       dp[i][j] = Math.min(dp[i-1][j], dp[i][j-1]) + grid[i][j]
>     }
>   }
> 
>   return dp[rows - 1][cols - 1]
> };
> ```
