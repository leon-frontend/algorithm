# 贪心算法理解

> **「贪心算法 greedy algorithm」是一种常见的 解决优化问题 的算法**
>
> - 其基本思想是在问题的每个决策阶段，都选择当前看起来最优的选择
> 
> - _==**即 贪心地做出局部最优的决策，以期获得全局最优解**==_
> 
> - 贪心算法简洁且高效，在许多实际问题中有着广泛的应用

> **贪心算法 vs 动态规划**
>
> - 贪心算法和动态规划都常用于 解决优化问题
> 
> - 它们之间存在一些相似之处，比如都依赖最优子结构性质，但工作原理不同
> 
> - 动态规划会根据之前阶段的所有决策来考虑当前决策，并使用过去子问题的解来构建当前子问题的解
> 
> - 贪心算法不会考虑过去的决策，而是一路向前地进行贪心选择，不断缩小问题范围，直至问题被解决

> **优点与局限性**
>
> - 贪心算法不仅操作直接、实现简单，而且通常效率也很高
> 
> - _==**然而，对于某些硬币面值组合，贪心算法并不能找到最优解**==_
> 
> - _**正例**_ ⇒ `**coins=[1,5,10,20,50,100]**`：在该硬币组合下，给定任意 _**amt**_，贪心算法都可以找到最优解。
> 
> - _**反例**_ ⇒ ==`**coins=[1,20,50]**`==：假设==`**amt=60**`==，贪心算法只能找到 _**50+1×10**_ 的兑换组合，共计 _**11**_ 枚硬币，但动态规划可以找到最优解 _**20+20+20**_，仅需 _**3**_ 枚硬币
> 
> - _**反例**_ ⇒ `**coins=[1,49,50]**`：假设`**amt=98**`，贪心算法只能找到 _**50+1×48**_ 的兑换组合，共计_**49**_ 枚硬币，但动态规划可以找到最优解 _**49+49**_，仅需 _**2**_ 枚硬币

> **贪心算法特性**
>
> - **贪心选择性质**：_==**只有当局部最优选择始终可以导致全局最优解时**==_，贪心算法才能保证得到最优解
> 
> - **最优子结构**：原问题的最优解包含子问题的最优解

> **贪心问题的解决步骤**
>
> 1. **问题分析**：梳理与理解问题特性，包括状态定义、优化目标和约束条件等。这一步在回溯和动态规划中都有涉及
> 
> 1. **确定贪心策略**：确定如何在每一步中做出贪心选择。这个策略能够在每一步减小问题的规模，并最终解决整个问题
> 
> 1. **正确性证明**：通常需要证明问题具有贪心选择性质和最优子结构。这个步骤可能需要用到数学证明，例如归纳法或反证法

# 分数背包问题

> **问题描述**
>
> > [!important] **[Hello 算法 - 分数背包问题](https://www.hello-algo.com/chapter_greedy/fractional_knapsack_problem/)**
>
> - 给定 **n** 个物品，第 **i** 个物品的重量为`**wgt[i-1]**`、价值为`**val[i-1]**`，和一个容量为 **cap** 的背包。每个物品只能选择一次，但可以选择物品的一部分，价值根据选择的重量比例计算。问在限定背包容量下背包中物品的最大价值。
> 
> ![[贪心算法 - 5.png|Untitled 89.png]]

> _**对物品任意地进行切分，并按照 重量比例 来计算相应价值**_
>
> - 对于物品 _**i**_ ，它在单位重量下的价值为`**val[i-1] / wgt[i-1]**` ，简称单位价值
> 
> - 假设放入一部分物品 _**i**_，重量为 _**w**_，则背包增加的价值为==`**w × val[i-1] / wgt[i-1]**`==
> 
> ![[贪心算法 - 6.png|Untitled 1 60.png]]

> _**贪心策略确定**_
>
> - 最大化背包内物品总价值，本质上是 最大化单位重量下的物品价值
> 
> - 将物品按照单位价值 从高到低 进行排序
> 
> - 遍历所有物品，_==**每轮贪心地选择 单位价值最高 的物品**==_
> 
> - 若剩余背包容量不足，则使用当前物品的一部分填满背包
> 
> ![[贪心算法 - 7.png|Untitled 2 56.png]]
>
> ```JavaScript
> /* 物品 */
> class Item {
>     constructor(w, v) {
>         this.w = w; // 物品重量
>         this.v = v; // 物品价值
>     }
> }
> 
> /* 分数背包：贪心 */
> function fractionalKnapsack(wgt, val, cap) {
>     // 创建物品列表，包含两个属性：重量、价值
>     const items = wgt.map((w, i) => new Item(w, val[i]));
>     // 按照单位价值 item.v / item.w 从高到低进行排序
>     items.sort((a, b) => b.v / b.w - a.v / a.w);
>     // 循环贪心选择
>     let res = 0;
>     for (const item of items) {
>         if (item.w <= cap) {
>             // 若剩余容量充足，则将当前物品整个装进背包
>             res += item.v;
>             cap -= item.w;
>         } else {
>             // 若剩余容量不足，则将当前物品的一部分装进背包
>             res += (item.v / item.w) * cap;
>             // 已无剩余容量，因此跳出循环
>             break;
>         }
>     }
>     return res;
> }
> ```

# 最大切分乘积问题

> **问题描述**
>
> > [!important] **[Hello 算法 - 最大切分乘积问题](https://www.hello-algo.com/chapter_greedy/max_product_cutting_problem/)**
>
> - 给定一个正整数 **n**，将其切分为至少两个正整数的和，求切分后所有整数的乘积最大是多少
> 
> ![[贪心算法 - 8.png|Untitled 3 42.png]]

> _**什么时候乘积会比原来的 n 大？**_
>
> - 给定一个正整数 **n，**当`**1 * (n - 1)**`时，乘积肯定比 **n** 小。只有当`**2 * (n - 2)**`时，乘积才可能比 **n** 大。
> 
> - _**什么情况下乘积****一定****比 n 大？⇒**_ 当 **n** 满足==`**2(n - 2) ≥ n**`==时，即==`**n ≥ 4**`==时，乘积一定比 **n** 大.
> 
> - 当`**n ≥ 4**`时，切分出一个 **2** 后乘积会变大，这说明大于等于 **4** 的整数都应该被切分

> _**贪心策略 1**_
>
> - 如果切分方案中包含`**≥ 4**`的因子，那么它就应该被继续切分，最终的切分方案只应出现 **1、2、3** 这三种因子。
> 
> - _**求解最优因子 ⇒**_ 在 **1、2、3** 这三个因子中，显然 **1** 是最差的，因为切分出 **1** 反而会导致乘积减小，所以当`**n ≥ 4时**`，尽量不要有 **1** 作为因子，即当余数为 **1** 时，由于`**2×2 > 1×3**`，因此应将最后一个 **3** 替换为 **2。**当`**n=6**`时，有`**3×3 > 2×2×2**`。==这意味着切分出== ==**3**== ==比切分出== ==**2**== ==更优。==

> _**贪心策略 2**_
>
> - 在切分方案中，最多只应存在两个 _**2**_
> 
> - _==**因为三个 2 总是可以替换为两个 3，从而获得更大的乘积**==_

> _**综上所述，可推理出以下贪心策略**_
>
> - 输入整数 _**n**_，从其不断地切分出因子 _**3**_，直至余数为 _**0、1、2**_
> 
> - 当余数为 _**0**_ 时，代表 _**n**_ 是 _**3**_ 的倍数，因此不做任何处理
> 
> - 当余数为 _**2**_ 时，不继续划分，保留
> 
> - 当余数为 _**1**_ 时，由于`**2×2 > 1×3**`，因此应将最后一个 _**3**_ 替换为 _**2**_

> _**边界情况**_
>
> - 对于`**n≤3**`的边界情况，必须拆分出一个 _**1**_，乘积为`**1 × (n-1)**`
> 
> - 因为要满足题目要求 ⇒ 将其切分为至少两个正整数的和

> _**代码实现**_
>
> ```JavaScript
> /* 最大切分乘积：贪心 */
> function maxProductCutting(n) {
>     // 当 n <= 3 时，必须切分出一个 1
>     if (n <= 3) {
>         return 1 * (n - 1);
>     }
>     // 贪心地切分出 3 ，a 为 3 的个数，b 为余数
>     let a = Math.floor(n / 3);
>     let b = n % 3;
>     if (b === 1) {
>         // 当余数为 1 时，将一对 1 * 3 转化为 2 * 2
>         return Math.pow(3, a - 1) * 2 * 2;
>     }
>     if (b === 2) {
>         // 当余数为 2 时，不做处理
>         return Math.pow(3, a) * 2;
>     }
>     // 当余数为 0 时，不做处理
>     return Math.pow(3, a);
> }
> ```

# 买卖股票的最佳时机

> _**问题描述**_
>
> > [!important] ** [LeetCode - 买卖股票的最佳时机](https://leetcode.cn/problems/best-time-to-buy-and-sell-stock/description/?envType=study-plan-v2&envId=top-100-liked) **
>
> - 给定一个数组 `prices` ，它的第 `i` 个元素 `prices[i]` 表示一支给定股票第 `i` 天的价格。你只能选择 **某一天** 买入这只股票，并选择在 **未来的某一个不同的日子** 卖出该股票。设计一个算法来计算你所能获取的最大利润。返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 `0`
> 
> ```JavaScript
> /**
>  * @param {number[]} prices
>  * @return {number}
>  */
> var maxProfit = function(prices) {
>   // 使用一个变量记录股票的最低价格
>   let minPrice = Infinity
> 
>   // 使用一个变量记录最大利润
>   let maxProfit = 0
> 
>   // 遍历数组 prices
>   for (let i = 0; i < prices.length; i++) {
>     if (prices[i] < minPrice) {
>       // 如果当前值比 股票的最低价格 要小，则买入股票
>       minPrice = prices[i]
>     }else {
>       // 如果当前值比 股票的最低价格 要大，则卖出股票，并记录最大值
>       maxProfit = Math.max(prices[i] - minPrice, maxProfit)
>     }
>   }
> 
>   return maxProfit
> };
> ```

# 跳跃游戏 I

> _**问题描述**_
>
> > [!important] ** [LeetCode - 跳跃游戏](https://leetcode.cn/problems/jump-game/description/?envType=study-plan-v2&envId=top-100-liked) **
>
> - 给你一个非负整数数组 `nums` ，你最初位于数组的 **第一个下标** 。数组中的每个元素代表你在该位置可以跳跃的最大长度。判断你是否能够到达最后一个下标，如果可以，返回 `true` ；否则，返回 `false` 。>

> _**算法分析**_
>
> - 从第一个位置开始，判断从当前位置可以到达的最远距离。如果最远距离大于等于数组的长度，则说明可以到达最后一个下标。否则，我们更新当前位置为 从当前位置可以到达的最远距离，继续判断。
> 
> ```JavaScript
> /**
>  * @param {number[]} nums
>  * @return {boolean}
>  */
> var canJump = function(nums) {
>   // 使用一个变量记录数组中元素可以跳跃的最远距离
>   // maxLen 可以理解为 最远距离 的下标
>   let maxLen = 0
> 
>   // 从第一个元素开始，逐个判断可以跳跃的最远距离
>   for (let i = 0; i < nums.length; i++) {
>     // 若 当前索引 大于 可以跳跃的最远距离，则说明不能跳跃到最后一个下标
>     if (i > maxLen) return false
>     // 若 当前索引 小于 可以跳跃的最远距离，更新最长跳跃距离
>     maxLen = Math.max(maxLen, i + nums[i])
>   }
> 
>   // 若没有返回 false，说明 当前索引 小于 可以跳跃的最远距离
>   // 循环结束时，当前索引 i === nums.length，且 maxLen > nums.length
>   // 所以可以跳跃到最后一个下标
>   return true
> }
> ```

# 跳跃游戏 II

> _**问题描述**_
>
> > [!important] ** [LeetCode - 跳跃游戏 II](https://leetcode.cn/problems/jump-game-ii/description/?envType=study-plan-v2&envId=top-100-liked) **
>
> - 给定一个长度为 `n` 的 **0 索引**整数数组 `nums`。初始位置为 `nums[0]`，每个元素 `nums[i]` 表示从索引 `i` 向前跳转的最大长度。换句话说，如果你在 `nums[i]` 处，你可以跳转到任意 `nums[i + j]` 处，其中 `0 <= j <= nums[i]`，`i + j < n`。返回到达 `nums[n - 1]` 的最小跳跃次数。生成的测试用例可以到达 `nums[n - 1]`。
> 
> 

> _**贪心算法**_
>
> - 从数组的第一个元素开始遍历，用一个变量 `**end**` 记录当前步骤能够到达的最远距离
> 
> - 使用一个变量 _**maxReach**_ 来记录在当前步骤及之前所有步骤中，能够达到的最远距离
>     
>     - 或者理解为下一次跳跃的最大距离
>     
> 
> - 当遍历到 `**end**` 时，意味着已经到达了当前步骤能达到的最远距离，此时需要进行一次跳跃，跳跃到 _**maxReach**_ 指示的最远位置
> 
> - 每当进行一次跳跃时，更新 `**end**` 为 _**maxReach**_ ，并增加跳跃次数

> _**边界情况**_
>
> - 在遍历数组时，_==**不访问最后一个元素**==_
> 
> - 这是因为在访问最后一个元素之前，我们的边界一定大于等于最后一个位置，否则就无法跳到最后一个位置了
> 
> - 如果访问最后一个元素，在边界正好为最后一个位置的情况下，我们会增加一次「不必要的跳跃次数」，因此我们不必访问最后一个元素

> _**代码详解**_
>
> ```JavaScript
> var jump = function(nums) {  
>   let jumps = 0 // 声明一个变量记录 跳跃步数  
>   let curReach = 0 // 声明变量记录当前步骤能达到的最远距离  
>   let maxReach = 0 // 声明变量记录 在当前步骤及之前所有步骤中 能够达到的最远距离
>   
>   // 循环遍历每一个元素，注意边界情况
>   // ###### 不访问最后一个元素 #####
>   // 如果访问最后一个元素，在边界正好为最后一个位置的情况下，
>   // 我们会增加一次「不必要的跳跃次数」，因此我们不必访问最后一个元素
>   for (let i = 0; i < nums.length - 1; i++) {
>     // 更新所有步骤中能到达的最远距离
>     maxReach = Math.max(maxReach, i + nums[i])
>     // 当到达 当前步骤 的最远距离时，需要进行一次跳跃
>     if (i === curReach) {
>       jumps++ // 进行一次跳跃
>       curReach = maxReach // 更新当前步骤中能到达的最远距离
>     }
>   }
>   return jumps
> };
> ```

# 划分字母区间

> **问题描述**
>
> > [!important] ** [LeetCode - 划分字母区间](https://leetcode.cn/problems/partition-labels/description/?envType=study-plan-v2&envId=top-100-liked) **
>
> - 给你一个字符串 s ，把这个字符串划分为尽可能多的片段，同一字母最多出现在一个片段中。注意，划分结果需要满足：将所有划分结果按顺序连接，得到的字符串仍然是 `s`。返回一个表示每个字符串片段的长度的列表
> 
> 

> **使用贪心算法动态更新当前片段的结束位置**
>
> - 建一个数组 `lastIndexOfChar`，用于记录每个字母在字符串中最后出现的位置。这个数组的索引表示字母在字母表中的索引（比如 `a` 对应索引 `0`，`b` 对应索引 `1`，以此类推），数组的值表示该字母最后出现的位置。
> 
> - 遍历字符串 `s`，更新 `lastIndexOfChar` 数组中每个字母的最后出现位置。
> 
> - 初始化两个指针 `start` 和 `end`，它们分别指向当前片段的起始位置和结束位置，初始值都 `0`。
> 
> - 再次遍历字符串 `s`，对于每个字符：
>     
>     - 更新当前片段的结束位置 `end`，取当前字符的最后出现位置与当前片段的结束位置的最大值
>     
>     - 如果当前位置等于当前片段的结束位置，则找到一个片段，将当前片段的长度（`end - start + 1`）添加到结果数组中，并更新 `start` 指针为下一个片段的起始位置。
>     
> 
> - 返回结果数组，其中每个元素表示一个片段的长度

> **代码详解**
>
> ```JavaScript
> /**
>  * @param {string} s
>  * @return {number[]}
>  */
> var partitionLabels = function(s) {
>   // 声明返回结果
>   const res = []
> 
>   // 声明一个数组，记录 s 中每个字符最后出现的索引
>   const lastIndex = Array(26)
> 
>   // 记录 s 中每个字符最后出现的索引
>   for (let i = 0; i < s.length; i++) {
>     // s.charCodeAt(i) 表示 s 中第 i 个索引元素的 ASCII 码
>     lastIndex[s.charCodeAt(i) - 'a'.charCodeAt(0)] = i
>   }
> 
>   // 由于同一字母最多出现在一个片段中，因此用 end 变量不断更新当前片段的结束位置
>   // 声明 start 表示当前片段的开始位置
>   let start = 0, end = 0
> 
>   for (let i = 0; i < s.length; i++) {
>     end = Math.max(end, lastIndex[s.charCodeAt(i) - 'a'.charCodeAt(0)])
> 
>     // 如果 当前遍历的位置 === 当前片段的结束位置 => 切分片段
>     if (i === end) {
>       res.push(i - start + 1) // 分片，加入结果数组
>       start = i + 1 // 更新 start 值，进行下一个分片操作
>     }
>   }
> 
>   return res
> };
> ```
