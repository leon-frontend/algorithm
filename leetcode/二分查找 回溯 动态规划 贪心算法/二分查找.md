# 1. 普通二分查找

> **理解**
>
> > [!important] **[Hello 算法 - 二分查找](https://www.hello-algo.com/chapter_searching/binary_search/)**
>
> - 二分查找 binary search 是一种基于分治策略的高效搜索算法
> - 它利用数据的有序性，每轮缩小一半搜索范围，直至找到目标元素或搜索区间为空为止

> **问题描述**
>
> - 给定一个长度为 n 的数组 nums ，元素按从小到大的顺序排列且不重复。请查找并返回元素 target 在该数组中的索引。若数组不包含该元素，则返回 −1。
> 
> ![[二分查找 - 1.png|Untitled 81.png]]

> **算法分析**
>
> - 使用`while (i <= j)`为循环条件 ⇒ 如果`i`和`j`相等，意味着搜索空间被缩减到仅剩一个元素，这个元素需要被检查，因为它可能就是我们要找的目标值。
> 
> - 计算中点索引 `m = ⌊(i + j) / 2⌋`，其中 `⌊⌋` 表示向下取整操作
> 
> - 判断 `nums[m]` 和 `target` 的大小关系，分为以下三种情况
>     
>     - 当 `nums[m] < target` 时，说明 `target` 在区间 `[m + 1, j]` 中，因此执行 `i = m + 1`
>     
>     - 当 `nums[m] > target` 时，说明 `**target**` 在区间 `[i , m - 1]` 中，因此执行 `j = m - 1`
>     
>     - 当 `nums[m] = target` 时，说明找到 `target` ，因此返回索引 `m`
>     
> 
> - 若数组不包含目标元素，搜索区间最终会缩小为空。此时返回 _**−1**_

> **使用双闭区间**
>
> ```JavaScript
> /* 二分查找（双闭区间） */
> function binarySearch(nums, target) {
>   // 初始化双闭区间 [0, n-1] ，即 i, j 分别指向数组首元素、尾元素
>   let i = 0,
>   let j = nums.length - 1;
>   // 循环，当搜索区间为空时跳出（当 i > j 时为空）
>   while (i <= j) {
>     // 计算中点索引 m ，使用 parseInt() 向下取整
>     const m = parseInt(i + (j - i) / 2); // 为了避免大数越界，不使用 (i + j) / 2
>     if (nums[m] < target)
>         // 此情况说明 target 在区间 [m+1, j] 中
>         i = m + 1;
>     else if (nums[m] > target)
>         // 此情况说明 target 在区间 [i, m-1] 中
>         j = m - 1;
>     else return m; // 找到目标元素，返回其索引
>   }
>   // 未找到目标元素，返回 -1
>   return -1;
> }
> ```

# 2. 二分查找插入点 1 & 无重复元素的情况

> Reference
>
> > [!important] **[Hello 算法 - 二分查找插入点](https://www.hello-algo.com/chapter_searching/binary_search_insertion/)**

> 问题描述
>
> - 给定一个长度为 **n** 的有序数组 **nums** 和一个元素 **target** ，数组不存在重复元素。现将 **target** 插入数组 **nums** 中，并保持其有序性。若数组中已存在元素 **target** ，则插入到其左方。请返回插入后 **target** 在数组中的索引。
> 
> ![[二分查找 - 2.png|Untitled 1 52.png]]

> 算法分析
>
> - 算法分析 1 ⇒ 当数组中 **包含 target** **时，插入点的索引是否是该元素的索引？**_
>     
>     - 将 _**target**_ 插入到相等元素的左边，这意味着新插入的 _**target**_ 替换了原来 _**target**_ 的位置。也就是说，当数组包含 **target** 时，插入点的索引就是该 **target** 的索引。
>     
> 
> - 算法分析 2 ⇒ 当数组中不存在 target 时，插入点是哪个元素的索引？
>     
>     - 当 `nums[m] < target` 时 **i** 移动，这意味着指针 _**i**_ 在向大于等于 **target** 的元素靠近。同理，指针 **j** 始终在向小于等于 **target** 的元素靠近。
>     
>     - 当二分查找结束时，**i** 指向首个大于 **target** 的元素，**j** 指向首个小于**target** 的元素。即当数组 不包含 **target** 时，插入索引为 **i**
>     
> 
> ```JavaScript
> /* 二分查找插入点（无重复元素） */
> function binarySearchInsertionSimple(nums, target) {
>     let i = 0,j = nums.length - 1, m // 初始化双闭区间 [0, n-1]
>     while (i <= j) {
>         m = Math.floor(i + (j - i) / 2); // 计算中点索引 m, 使用 Math.floor() 向下取整
>         if (nums[m] < target) {
>             i = m + 1; // target 在区间 [m+1, j] 中
>         } else if (nums[m] > target) {
>             j = m - 1; // target 在区间 [i, m-1] 中
>         } else {
>             return m; // 找到 target ，返回插入点 m
>         }
>     }
>     // 未找到 target ，返回插入点 i
>     return i;
> }
> ```

# 3. 二分查找插入点 2 & 存在重复元素的情况

> **问题描述**
>
> - 给定一个长度为 _**n**_ 的有序数组 _**nums**_ 和一个元素 _**target**_ ，数组可能存在重复元素
> 
> - 现将 _**target**_ 插入数组 _**nums**_ 中，并保持其有序性
> 
> - 若数组中已存在元素 _**target**_ ，则插入到其 左方
>     
>     - _==**需要查找数组中最左一个 target 的索引**==_
>     
> 
> - 请返回插入后 _**target**_ 在数组中的索引

> **算法分析**
>
> - 假设数组中存在多个 _**target**_ ，则普通二分查找只能返回其中一个 _**target**_ 的索引，而无法确定该元素的 左边和右边 还有多少 _**target**_
> 
> - 如果 target 存在于数组 nums 中，那么最终 i 指向的位置就是 target 第一次出现的位置
> 
> - 如果 target 不存在于数组 nums 中,那么最终 i 指向的就是 target 应该插入的位置：即 **i** 指向首个大于 **target** 的元素
> 
> ![[二分查找 - 3.png|Untitled 2 48.png]]
>
> ```JavaScript
> /* 二分查找插入点（存在重复元素） */
> function binarySearchInsertion(nums, target) {
>     let i = 0,
>         j = nums.length - 1; // 初始化双闭区间 [0, n-1]
>     while (i <= j) {
>         const m = Math.floor(i + (j - i) / 2); // 计算中点索引 m, 使用 Math.floor() 向下取整
>         if (nums[m] < target) {
>             i = m + 1; // target 在区间 [m+1, j] 中
>         } else if (nums[m] > target) {
>             j = m - 1; // target 在区间 [i, m-1] 中
>         } else {
>             j = m - 1; // 首个小于 target 的元素在区间 [i, m-1] 中
>         }
>     }
>     // 返回插入点 i
>     return i;
> }
> ```

# 4. 二分查找边界

> **Reference**
>
> > [!important] **[Hello 算法 - 二分查找边界](https://www.hello-algo.com/chapter_searching/binary_search_edge/)**

> **情况 1 ⇒ 查找左边界**
>
> - 回忆二分查找插入点的方法，搜索完成后 **i** 指向最左一个**target**，查找插入点本质上是在查找最左一个 **target** 的索引
> 
> - 数组中可能不包含 **target** ，这种情况可能导致以下两种结果：_**1.**_ 插入点的索引 **i** 越界；_**2.**_ 元素 **nums[i]** 与 **target** 不相等。当遇到以上两种情况时，直接返回 **−1** 即可。
> 
> ![[二分查找 - 4.png|Untitled 3 35.png]]
>
> ```JavaScript
> /* 二分查找最左一个 target */
> function binarySearchLeftEdge(nums, target) {
> 	/* 二分查找插入点（存在重复元素） */
> 	function binarySearchInsertion(nums, target) {
>     let i = 0, j = nums.length - 1; // 初始化双闭区间 [0, n-1]
>     while (i <= j) {
>         const m = Math.floor(i + (j - i) / 2); // 计算中点索引 m, 使用 Math.floor() 向下取整
>         if (nums[m] < target) {
>             i = m + 1; // target 在区间 [m+1, j] 中
>         } else if (nums[m] > target) {
>             j = m - 1; // target 在区间 [i, m-1] 中
>         } else {
>             j = m - 1; // 首个小于 target 的元素在区间 [i, m-1] 中
>         }
>     }
>     // 返回插入点 i
>     return i;
> 	}
> 
>   // 等价于查找 target 的插入点
>   const i = binarySearchInsertion(nums, target);
>   // 未找到 target ，返回 -1
>   if (i === nums.length || nums[i] !== target) return -1
>   // 找到 target ，返回索引 i
>   return i;
> }
> ```

> **情况 2 ⇒ 查找左边界**
>
> - _==**核心思想**==_ _==**⇒**==_ ==将查找最右一个== ==**target**== ==转化为查找最左一个====`**target + 1**`==
> 
> - 查找完成后，指针 **i** 指向最左一个 **target + 1**（如果存在），而 **j** 指向最右一个 **target** ，因此返回 **j** 即可
> 
> - ==若== ==**target**== ==不存在，====**i**== ==指向首个大于== ==**target**== ==的元素，====**j**== ==指向首个小于== ==**target**== ==的元素==
> 
> ![[二分查找 - 5.png|Untitled 4 27.png]]
>
> ```JavaScript
> /* 二分查找最右一个 target */
> function binarySearchRightEdge(nums, target) {
>   // 转化为查找最左一个 target + 1
>   const i = binarySearchInsertion(nums, target + 1);
>   // j 指向最右一个 target ，i 指向首个大于 target 的元素
> 	// 返回的插入点是 i ，因此需要将 i - 1 ，从而获得 j
>   const j = i - 1;
>   // 未找到 target ，返回 -1
>   if (j === -1 || nums[j] !== target) {
>       return -1;
>   }
>   // 找到 target ，返回索引 j
>   return j;
> }
> ```
# 5. 排序数组中查找元素

> **问题描述**
>
> > [!important] **[LeetCode - 在排序数组中查找元素的第一个和最后一个位置](https://leetcode.cn/problems/find-first-and-last-position-of-element-in-sorted-array/description/?envType=study-plan-v2&envId=top-100-liked)**
>
> - 给你一个按照非递减顺序排列的整数数组 `nums`，和一个目标值 `target`。请你找出给定目标值在数组中的开始位置和结束位置。如果数组中不存在目标值 `target`，返回 `[-1, -1]`。你必须设计并实现时间复杂度为 `O(log n)` 的算法解决此问题。
> 
> ```JavaScript
> /**
>  * @param {number[]} nums
>  * @param {number} target
>  * @return {number[]}
>  */
> var searchRange = function(nums, target) {
>   // 声明返回结果变量
>   let start = -1, end = -1
> 
>   // 二分查找：查找 target 的第一个位置
>   const findStart = (nums, target) => {
>     // 声明左右指针和 mid
>     let left = 0, right = nums.length - 1, mid
> 
>     while (left <= right) {
>       mid = Math.floor(left + (right - left) / 2)
>       if (nums[mid] < target) left = mid + 1
>       else right = mid - 1
>     }
> 
>     // 使用“循环二分查找”，
>     // 若 target 存在，则最后 left 的位置就是最左边的 target
>     // 若 target 不存在，则最后 left 的位置就是首个大于 target 的位置
>     return left
>   }
> 
>   // 第一次二分查找：查找 target 的第一个位置
>   start = findStart(nums, target)
>   // 第二次二分查找：查找 target + 1 的第一个位置，即 target 最右边的位置 + 1
>   // 返回的插入点是 left ，因此需要将 left - 1 ，从而获得 right
>   end = findStart(nums, target + 1) - 1
> 
>   // 判断 start 和 end 的值是否符合需求
>   if (start > end) return [-1, -1]
>   else return [start, end]
> };
> ```

# 6. 搜索旋转排序数组

> _**问题描述**_
>
> > [!important] **[LeetCode - 搜索旋转排序数组](https://leetcode.cn/problems/search-in-rotated-sorted-array/description/?envType=study-plan-v2&envId=top-100-liked)**
>
> - 整数数组 **nums** 按升序排列，数组中的值互不相同。在传递给函数之前，**nums** 在预先未知的某个下标 **k** `( 0 <= k < nums.length)`上进行了 旋转，使数组变为`[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`（下标从 **0** 开始计数）。
> 
> - 例如， `[0,1,2,4,5,6,7]`在下标 **3** 处经旋转后可能变为`[4,5,6,7,0,1,2]` 。
> 
> - 给你旋转后的数组 **nums** 和一个整数 **target**，如果 **nums** 中存在这个目标值 **target**，则返回它的下标，否则返回`-1`。你必须设计一个时间复杂度为`O(log n)`的算法解决此问题。
> 

> **局部有序数组进行二分查找**
>
> - 数组本身不是有序的，进行旋转后只保证了数组的局部是有序的，可以在局部有序数组中进行二分查找。将数组从中间分开成左右两部分的时候，一定有一部分的数组是有序的。
> 
> - 拿示例来看，我们从 _**6**_ 这个位置分开以后数组变成了`**[4, 5, 6]**`和`**[7, 0, 1, 2]**`两个部分，其中左边`**[4, 5, 6]**`这个部分的数组是有序的，其他也是如此。

> **寻找局部有序数组 & 代码实现**
>
> - **情况 1** ⇒ 如果`[l, mid - 1]`是有序数组，且 _**target**_ 的大小满足`[nums[l],nums[mid]]`，则我们应该将搜索范围缩小至 `[l, mid - 1]`，否则在 `[mid + 1, r]` 中寻找。
> 
> - **情况 2** ⇒ 如果 `[mid, r]` 是有序数组，且 target 的大小满足 `[nums[mid+1],nums[r]]`，则我们应该将搜索范围缩小至 `[mid + 1, r]`，否则在 `[l, mid - 1]` 中寻找
> 
> - mid 的左右两侧数组，至少有一侧是有序的
> 
> ![[二分查找 - 8.png|Untitled 2 49.png]]
>
> ```JavaScript
> /**
>  * @param {number[]} nums
>  * @param {number} target
>  * @return {number}
>  */
> var search = function(nums, target) {
>   // 定义左右指针
>   let left = 0, right = nums.length - 1, mid
> 
>   // 二分查找
>   while (left <= right) {
>     mid = Math.floor(left + (right - left) / 2)
> 
>     // 若找到 target 则直接返回索引
>     if (nums[mid] === target) return mid
> 
>     // 判断 mid 的左侧数组是否有序
>     if (nums[left] <= nums[mid]) {
>       // 判断 target 是否在左侧有序数组中
>       // 若 target 在左侧数组中，则向左移动 right；反之向右移动 left
>       if (nums[left] <= target && target < nums[mid]) right = mid - 1
>       else left = mid + 1
> 
>     } else { // mid 左右两侧至少有一个是有序数组，若左侧不是有序的，则右侧必是有序的
>       // 判断 target 是否在右侧有序数组中
>       // 若 target 在右侧数组中，则向右移动 left；反之向左移动 right
>       if (nums[mid] < target && target <= nums[right]) left = mid + 1
>       else right = mid - 1
>     }
>   }
> 
>   return -1
> };
> ```

# 7. 寻找旋转排序数组中的最小值

> _**问题描述**_
>
> > [!important] **[LeetCode - 寻找旋转排序数组中的最小值](https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/description/?envType=study-plan-v2&envId=top-100-liked)**
>
> - 已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次旋转后，得到输入数组。
> 
> - 例如，原数组`**nums = [0,1,2,4,5,6,7]**`在变化后可能得到：
>     
>     - 若旋转 `4` 次，则可以得到 `[4,5,6,7,0,1,2]`
>     
>     - 若旋转 `7` 次，则可以得到 `[0,1,2,4,5,6,7]`
>     
> 
> - 注意，数组 `[a[0], a[1], a[2], ..., a[n-1]]` **旋转一次** 的结果为数组 `[a[n-1], a[0], a[1], a[2], ..., a[n-2]]` 。
> 
> - 给你一个元素值互不相同的数组 `nums` ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的最小元素 。你必须设计一个时间复杂度为 `O(log n)` 的算法解决此问题。

> **一个不包含重复元素的升序数组在经过旋转之后，可以得到下面可视化的折线图**
>
> - **被旋转到前面的元素一定比后面的元素大**
> 
> - 其中横轴表示数组元素的下标，纵轴表示数组元素的值
> 
> - 图中标出了最小值的位置，是我们需要查找的目标
> 
> ![[二分查找 - 10.png|Untitled 4 28.png]]

> _**考虑数组中的最后一个元素 x**_
>
> - 在最小值右侧的元素（不包括最后一个元素本身），它们的值一定都严格小于 x 。而在最小值左侧的元素，它们的值一定都严格大于 x 。因此，我们可以根据这一条性质，通过二分查找的方法找出最小值。

> **将中轴元素 nums[mid] 与右边界元素 nums[right] 进行比较，可能会有以下的三种情况**
>
> - _==**使用右指针指向最小值，不断的迭代 right**==_
> 
> - 第一种情况是 `nums[mid] < nums[right]` ⇒ 说明 **nums[mid]** 是最小值右侧的元素，也可能是最小值。因此可以忽略二分查找区间的右半部分。
> 
>     ![[二分查找 - 11.png|Untitled 5 22.png]]
>     
> 
> - 第二种情况是==`**nums[mid] > nums[right]**`== ==⇒== 说明 **nums[pivot]** 必是最小值左侧的元素，因此可以忽略二分查找区间的左半部分。
>
>     ![[二分查找 - 12.png|Untitled 6 17.png]]
>     
> 
> - 由于数组不包含重复元素，并且只要当前的区间长度不为 1，mid 就不会与 right 重合；而如果当前的区间长度为 **1**，这说明我们已经可以结束二分查找了。因此不会存在 `nums[mid] = nums[right]` 的情况。当二分查找结束时，我们就得到了最小值所在的位置。

> _**代码实现**_
>
> ```JavaScript
> var findMin = function(nums) {
>   let left = 0, right = nums.length - 1, mid
> 
>   // 二分查找
>   // 疑问：为什么while的条件是low<high,而不是low<=high呢
>   // 解答：low<high，假如最后循环到{*,10,1,*}的这种情况时，nums[low]=10,nums[high]=1,nums[mid]=10,low=mid+1,
>   // 直接可以跳出循环了,所以low<high,此时low指向的就是最小值的下标;
>   // 如果low<=high的话，low=high，还会再不必要的循环一次，此时最后一次循环的时候会发生low==high==mid，
>   // 则nums[mid]==nums[high]，则会走一次else语句，则low=mid+1,此时low指向的是最小值的下一个下标，
>   // 则需要return[low-1]
>   // 新理解 => 如果有等号，在最后一次判断时会陷入死循环
>   while (left < right) {
>     mid = Math.floor(left + (right - left) / 2)
>     if (nums[mid] <= nums[right]) {
>       // 此时说明 nums[mid] 是最小值右侧的元素，即最小值在 mid 左侧
>       // 则在 mid 左侧查找最小值，right 左移
>       // 疑问：为什么 high = mid;而不是 high = mid-1;
>       // 解答：{4,5,1,2,3}，如果high=mid-1，则丢失了最小值1
