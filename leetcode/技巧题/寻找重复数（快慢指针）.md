### _问题描述_

> 给定一个包含 `n + 1` 个整数的数组 `nums` ，其数字都在 `[1, n]` 范围内（包括 `1` 和 `n`），可知至少存在一个重复的整数。
>
> 假设 `nums` 只有 **一个重复的整数** ，返回 **这个重复的数** 。
>
> 你设计的解决方案必须 **不修改** 数组 `nums` 且只用常量级 `O(1)` 的额外空间。

---

### _算法分析_

> _**使用快慢指针的思想来解决，类似于判断链表中是否有环的问题，等价于 环形链表 Ⅱ**_
>
> > [!important]

> _==**将数组中的数字看作是一个链表，每个数字都对应一个下标 ⇒ i → nums[i]**==_
>
> - 根据题目描述，数字的范围是 `**[1, n]**`，所以数组中的每个数字都可以看作是一个指向下一个数字的指针。
>     
>     - `**[1,3,4,2,2]**`可以理解为`**0 —> 1 —> 3 —> 2 <—> 4**`
>     
> 
> - 而且因为至少存在一个重复的整数，所以这个链表必然存在环
> 
> - 使用快慢指针，快指针每次移动两步，慢指针每次移动一步，直到它们相遇。
> 
> - 当快慢指针相遇时，将慢指针重新指向数组的起始位置，然后快慢指针同时以每次移动一步的速度向前移动，直到它们再次相遇。
> 
> - 当它们再次相遇时，就是链表中的环的入口，也就是重复的数字。
> 
> - 详解请看 _**环形链表 Ⅱ**_

> _**Code**_
>
> ```JavaScript
> /**
>  * @param {number[]} nums
>  * @return {number}
>  */
> var findDuplicate = function(nums) {
>   // 声明快慢指针 fast 走两步，slow 走一步
>   let fast = 0, slow = 0
> 
>   // 第一次相遇
>   do {
>     fast = nums[nums[fast]]
>     slow = nums[slow]
>   } while (fast !== slow)
> 
>   // 慢指针重新指向起始位置
>   slow = 0
> 
>   // 快慢指针以相同速度移动，直到第二次相遇
>   while (fast != slow) {
>     fast = nums[fast]
>     slow = nums[slow]
>   }
>   
>   return slow
> };
> ```
